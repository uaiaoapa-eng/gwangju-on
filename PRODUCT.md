# 광주ON — 상품 기획서

> 광주광역시 공무원과 시민이 AI(Claude)를 기반으로 함께 쓰는 **통합 행정 · 시민참여 플랫폼** 기획 문서.
> 본 저장소의 사이트는 이 기획을 시연하는 프로토타입(데모)이다.

## 1. 배경과 문제 정의

| 이해관계자 | 현재의 불편 |
|---|---|
| 시민 | 민원 창구가 분산(전화·방문·홈페이지·국민신문고)되어 "어디에 물어봐야 하는지"부터 어려움 |
| 공무원 | 반복 민원 응대에 시간 소모, 데이터는 있어도 시민용 콘텐츠로 가공할 시간·도구 부족 |
| 시 정책부서 | 시민 의견 수렴이 공청회·설문 등 단발성 채널에 머물러 상시적 참여 구조 부재 |

**핵심 가설**: 하나의 창구에서 ① AI가 민원을 즉시 안내하고 ② 시민이 제안·투표로 참여하며 ③ 행정이 데이터를 투명하게 공개하면, 민원 처리 비용은 줄고 정책 신뢰는 올라간다.

## 2. 제품 정의

- **제품명**: 광주ON (빛고을 광주가 켜진다 · On-line · 常時 ON)
- **한 줄 정의**: 묻고(AI 민원) → 제안하고(시민참여) → 함께 결정하는(정책투표) 광주 통합 시민 플랫폼
- **타깃**: 광주 시민(1차), 광주시·5개 자치구 공무원(2차)

## 3. 6대 핵심 서비스

1. **AI 민원 도우미** — Claude 기반 24시간 챗봇. 민원 내용을 자연어로 받아 담당 부서·처리 절차·필요 서류를 즉시 안내하고, 접수까지 연결. 반복 민원(쓰레기 배출, 주정차, 보조금 등) 1차 응대 자동화.
2. **시민 제안 · 공감** — 정책 아이디어를 등록하면 시민 공감을 거쳐 일정 기준(예: 공감 100) 도달 시 담당 부서가 30일 내 공식 답변.
3. **정책 라이브 투표** — 주민참여예산·정책 우선순위를 상시 투표로 결정. 결과는 실시간 공개.
4. **행정 투명 대시보드** — 민원 접수·처리율, 예산 집행 현황을 실시간 공개.
5. **AI 콘텐츠 스튜디오(공무원용)** — 행정 데이터·보도자료를 넣으면 카드뉴스·쉬운 요약문·SNS 문안을 AI가 생성. 부서별 홍보 제작 시간 단축.
6. **우리동네 광주** — 동구·서구·남구·북구·광산구 자치구별 이슈 게시판과 소식.

## 4. 단계별 로드맵

| 단계 | 기간(안) | 내용 |
|---|---|---|
| MVP(본 데모) | — | 정적 프로토타입: AI 챗 데모, 제안/공감, 라이브 투표, 대시보드 UI |
| 1차 오픈 | +3개월 | Claude API 연동 실 챗봇, 회원(간편인증), 제안 DB화, 국민신문고 연계 |
| 2차 | +6개월 | 공무원용 콘텐츠 스튜디오, 부서 답변 워크플로, 자치구 계정 |
| 3차 | +12개월 | 주민참여예산 정식 투표(본인인증), 공공데이터 개방 API |

## 5. 기술 구조(안)

- 프론트: 정적 웹(본 데모) → 차후 Next.js
- AI: Claude API (민원 분류·안내 = 도구 호출로 부서 DB 조회, 콘텐츠 생성 = 템플릿 프롬프트)
- 인증: 간편인증(PASS·카카오) / 공무원은 시 내부 SSO
- 데이터: 민원 통계·예산은 공공데이터포털 및 시 내부 시스템 연계

## 6. 성과 지표(KPI)

- 단순·반복 민원의 AI 1차 처리율 (목표 40%)
- 제안 → 부서 공식답변 도달 건수 / 답변 소요일
- 월간 참여 시민 수(MAU), 투표 참여율
- 공무원 콘텐츠 제작 시간 절감(제작 건당 시간)

## 7. 원칙

- **투명**: 처리 현황·통계는 기본 공개
- **책임**: AI 안내는 참고용임을 명시, 최종 책임 부서 항상 표기
- **접근성**: 쉬운 언어, 모바일 우선, 고령층 큰 글씨 모드
- 본 데모의 수치·응답은 모두 **시연용 가상 데이터**임
